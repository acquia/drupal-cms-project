# Frequently asked questions

## Is Local ready to go live?

No. Local is a demonstration template. Replace the demo content, apply your branding, and configure before launching.

## Is this the same as projects like LocalGov Drupal?

No. [LocalGov Drupal (LGD)](https://localgovdrupal.org/) is an established Drupal distribution with comprehensive content types, patterns, features
and governance. Local is a limited demonstration for Drupal CMS with Canvas.

## Do I need Drupal CMS, or can I use plain Drupal?

Local is designed for Drupal CMS, which bundles Canvas. Drupal CMS is the supported base.

## Can I use a different theme?

Yes. However, the Site template is bundled with the Consistent theme in an opiniated way.

If you would like to 'go custom', please follow the guidance at - [Canvas SDC components documentation](https://project.pages.drupalcode.org/local/customisation):

## Does Local support multiple languages?

No — this is a current limitation of Canvas. See the [Canvas multilingual meta issue](https://www.drupal.org/project/canvas/issues/3551464) for current status.

## How do I add new services?

1. Create a Service landing page at **Content → Add content → Service landing page**
2. Add a menu link at **Structure → Menus → Services**
3. Create Service pages with the new landing page set as their parent

The ECA workflow adds each Service page to the section listing automatically.

## Can I remove the demo content?

Yes. When creating your own content, delete demo nodes at **Content**, demo media at **Content → Media**, and demo menu links at **Structure → Menus**. Canvas pages are at **Content → Pages** — you could edit the Home page rather than delete it.

## Where do I report bugs?

Use the [issue queue on drupal.org](https://www.drupal.org/project/issues/local). Please search before filing to avoid duplicates.

## Where can I get help?

- As well as the project issue queues for Local to report problems, for general support [Drupal Slack](https://www.drupal.org/slack) is recommended.
